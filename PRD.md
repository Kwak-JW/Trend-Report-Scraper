# 트렌드 리포트 자동 수집기 (Trend Report Auto-Scraper) - PRD

## 1. 개요 (Product Overview)
본 제품은 다양한 웹사이트에 분산되어 있는 '트렌드 리포트'를 자동으로 감지하여 사용자의 로컬 PC에 PDF 또는 TXT 파일 형태로 저장해 주는 웹 기반 자동화 애플리케이션입니다.

## 2. 목적 (Objectives)
- **시간 절약:** 매번 여러 사이트를 방문해 리포트를 확인하고 다운로드하는 반복 업무를 제거.
- **범용적 접근:** 특정 사이트의 HTML 구조(Class 이름 등)에 종속되지 않고 'Heuristic(경험적 추론)' 기반으로 날짜, 첨부파일을 감지하여 다양한 사이트에 대응.

## 3. 핵심 요구사항 (Core Requirements)
1. **타겟 사이트 관리:** 사용자가 스크래핑할 대상 웹사이트 URL을 입력/추가/수정/삭제 가능.
2. **기간 설정:** 다운로드할 리포트의 기간(시작일~종료일)을 달력 입력기로 설정 가능.
3. **로컬 경로 지정:** 다운로드 받은 파일들이 저장될 개발자/사용자의 로컬 PC 폴더 경로 지정.
   - *클라우드 제약 보안:* 웹 환경 원격 실행 시를 대비하여 전체 파일을 ZIP으로 묶어주는 기능 지원.
4. **자동화 스크래핑 및 다운로드:**
   - Heuristics: YYYY-MM-DD 형태의 날짜 매칭으로 대상 게시물 식별.
   - 텍스트가 가장 긴 `<a>` 태그를 리포트 상세 페이지로 인식.
   - 다운로드 링크 감지: `.pdf` 포함 또는 '다운', 'pdf', '첨부', '원문' 키워드 포함 시 해당 링크 추출 후 파일 다운로드.
5. **Fall-back 구조:** PDF 첨부파일 등을 찾지 못할 경우 렌더링된 상세 페이지 본문과 이미지를 직접 `.pdf` 문서로 변환 저장.
6. **실시간 관제 (작업 모니터링):**
   - 현재 어느 사이트를 조회 중인지, 문서 다운로드가 성공했는지, 실패 후 텍스트 스크래핑으로 전환했는지를 로깅.
   - SSE(Server-Sent Events)를 통해 UI에 진행 과정을 실시간 스트리밍.
7. **작업 완료 알림:** UI 상에 "모든 작업 완료" 시스템 메시지 표시 및 완료 시점의 다운로드 버튼 노출.

## 4. 기술 스택 (Tech Stack)
- **Frontend:** React, Tailwind CSS, Vite (UI 제공 및 설정, 모니터링 뷰)
- **Backend:** Node.js (Express)
- **Crawling/Scraping:** Puppeteer (DOM 렌더링 및 인터셉트), Cheerio (빠른 HTML 파싱 및 DOM 탐색), Axios (파일 스트림 다운로드)
- **Utilities:** date-fns (날짜 처리), archiver (ZIP 압축)

## 5. 업데이트 기록 (Changelog)
- **v1.0.0:** 초기 요구사항 분석, Full-stack 환경 세팅, Regex 및 Heuristics 기반 크롤링 로직 구현, 웹소켓(SSE) 기반 실시간 로그 모니터링 UI 구축 완료.
- **v1.1.0:** KDI 등 특수 환경 대응(상세 페이지 fallback 검증, onclick 버튼 대응 등). HTTP 헤더를 통한 원본 파일명 유지 추가.
- **v1.2.0:**
  - 다운로드 파일 이름 규칙을 '서버측 난수 원본 파일명'에서 **직관적인 '리포트 제목(Report Title)' 기준**으로 롤백 및 고도화.
  - 지나친 치환식(`replace`)으로 인해 파일명에서 띄어쓰기가 사라지는 현상 방지. 특수문자만 치환하도록 `safeTitle` 로직 재조정.
  - 리스트에서 가져온 리포트 제목이 부실할 경우 상세 페이지의 `<title>` 태그를 교차 참조하도록 제목 추출 강건성 추가.
- **v1.4.0:** KDI(한국개발연구원) 사이트 최적화 및 메인 UI 편의성 개선
  - KDI `topicList` (경제동향, 거시/금융, 산업/무역) 카테고리 지원 및 페이지네이션(`pg`) 파라미터 호환 적용.
  - 파일 다운로드 시 응답 헤더(`Content-Disposition`) 정보를 추출하여 PDF, HWP, ZIP 등 정확한 원본 확장자가 동적으로 유지되도록 개선.
  - UI 상에 대상 Target sources 기본값으로 자주 쓰는 KIET, KDI 등의 URL을 8종 일괄 제공하고, UI에서 출처를 배지(Badge) 뱃지로 시각화.
- **v1.6.0 (Current):** 하드코딩된 특정 사이트 의존성 제거 및 완벽한 범용(Generic) 크롤링 엔진 구축
  - 특정 사이트를 판별하는 하드코딩 로직(`isKIET`, `isKDI`, `isKOTRA` 등)을 전면 제거하고, 어떤 URL이 입력되더라도 자동으로 페이징 파라미터(`page`, `pg`, `pageIndex` 등)를 탐색하여 순회하도록 범용 알고리즘으로 정리.
  - 링크 추출 로직을 정규화하여 `href`, `onclick(location.href)`, `onclick(window.open)` 등 표준적인 JS 이동을 포괄적으로 감지하고 다운로드 링크도 동일한 범용 규칙으로 탐색.
  - 불필요한 테스트 파일(`test-kotra.ts`, `kotra.html` 등) 및 임시 파일 삭제 등 구조 정리.
- **v1.3.0:** 산업연구원 (KIET) 전용 페이징 및 사이트 특화 최적화
  - 페이징 파라미터(`pg=x`)를 추적해 자동으로 과거 페이지를 찾아가는 순회(Pagination) 로직 도입 (최대 30 페이지).
  - '중국산업경제브리프' 아코디언 메뉴 대체: URL 파라미터(`year/month`) 기반 날짜 조합 우회 크롤링 기법 도입.
  - 날짜 형식 보강: `YYYY. MM월`, `YYYY년 MM월` 등 한글 혼합 날짜 포맷 인식 기능(`extractKoreanDateFallback`) 추가.
  - 동영상 보고서 필터: KIET 영상 보고서는 스크랩하지 않도록 텍스트 필터 예외 처리 추가.
