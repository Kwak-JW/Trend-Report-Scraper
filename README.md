# Trend Report Auto-Scraper 가이드

이 프로젝트는 지정된 웹사이트들을 순회하여 기간 내 작성된 트렌드 리포트를 자동으로 색인하고 로컬 환경에 PDF 혹은 TXT 형태의 리포트로 추출해주는 스크래핑 도구입니다.

## ⚠️ 클라우드 환경 vs 로컬 환경
이 코드는 AI Studio(클라우드 샌드박스) 환경 및 로컬 PC 환경 양쪽에서 모두 안전하게 작동하도록 풀스택(Full-stack) 아키텍처로 구현되었습니다.
- **클라우드 구동 시:** 환경적 브라우저 제약이 발생할 수 있으며, 다운로드 파일은 1차적으로 서버(컨테이너)의 지정 경로에 저장되고 작업 종료 후 `.zip` 확장자로 묶여 웹 브라우저를 통해 다운로드됩니다.
- **로컬 구동 시 (권장):** 사용자가 지정한 `C:\downloads\reports` 등 **자신의 실제 로컬 경로**에 수집된 파일이 직접 작성/저장됩니다.

## 🚀 로컬 PC에서 실행하는 방법 (단계별 안내)

로컬 PC에서 가장 완벽하게 작동합니다. 다음 단계를 순서대로 진행해 주세요.

### 1단계: 필수 소프트웨어 설치
PC에 **Node.js**가 설치되어 있어야 합니다.
- [Node.js 공식 홈페이지](https://nodejs.org/)에서 LTS(Long Term Support) 버전을 다운로드하고 설치하세요.

### 2단계: 프로젝트 다운로드 및 압축 해제
1. 현재 이 화면 우측 상단의 메뉴를 통해 **Export to ZIP** 또는 **GitHub 연동**을 통해 본 프로젝트의 소스 코드를 로컬 PC로 다운로드 받습니다.
2. 다운로드 받은 ZIP 파일의 압축을 원하시는 폴더에 풉니다.

### 3단계: 필수 패키지 설치
1. 압축을 푼 프로젝트 폴더에서 터미널(또는 명령 프롬프트, PowerShell)을 엽니다.
2. 아래 명령어를 입력하여 크롤링과 서버 구동에 필요한 모든 의존성 패키지를 설치합니다.
   ```bash
   npm install
   ```
   *(설치되는 주요 백엔드 패키지: `puppeteer`, `cheerio`, `archiver`, `axios`, `express`, `cors`, `date-fns` 등)*

### 4단계: 애플리케이션 실행
패키지 설치가 완료되었다면, 터미널에 다음 명령어를 입력합니다.

```bash
npm run dev
```

터미널에 `Server running on http://localhost:3000` 문구가 출력되면 정상적으로 실행된 것입니다.

### 5단계: 스크래핑 이용하기
1. 웹 브라우저(Chrome 권장)를 열고 `http://localhost:3000` 으로 접속합니다.
2. **타겟 URL 설정:** 스크래핑하고자 하는 대상 사이트 URL을 입력하고 `+` 버튼으로 추가합니다. (기본 예시 제거 가능)
3. **기간 설정:** 리포트가 업로드된 시작일과 종료일을 지정합니다.
4. **저장 폴더 경로:** 수집한 리포트 파일을 저장할 내 PC 안의 로컬 경로를 적습니다 (예: 윈도우라면 `C:\Users\Username\Downloads\Reports`, 맥이라면 `/Users/Username/Downloads/Reports`).
5. **자동 스크래핑 시작** 버튼을 누릅니다.
6. 우측 '작업 로그 & 모니터링' 창에서 브라우저가 보이지 않는 상태(Headless)로 각 사이트를 분석하고 수집하는 과정을 실시간으로 확인합니다.
7. 완료 알림이 나타나면 지정했던 로컬 폴더를 열어 PDF 또는 TXT 파일들이 정상적으로 들어있는지 확인합니다.

## 개발 및 동작 원리 (휴리스틱)
본 크롤러는 특정 사이트 레이아웃에 의존하지 않는 범용적 휴리스틱(Heuristic) 로직을 채택했습니다:
- **날짜 감지:** 화면에 노출된 텍스트 중 `YYYY-MM-DD` 또는 `YYYY.MM.DD` 패턴을 추출하여 필터링합니다. 목록에서 날짜를 찾지 못한 경우 해당 항목들을 클릭하여 상세 페이지에서 날짜의 유효성을 2차 검증합니다 (v1.1 업데이트).
- **제목/클릭 링크 추출:** 필터링된 행(tr, li) 내부에서 가장 텍스트가 긴 `<a>` 태그 또는 `onclick` 함수가 걸려있는 버튼을 본문 링크로 인식합니다.
- **첨부파일 탐색:** 상세 페이지 접속 후 `href`에 `.pdf`가 포함되어 있거나 버튼 이름에 `다운, pdf, 첨부, 원문`이라는 단어가 들어간 앵커/버튼 태그를 다운로드 링크로 특정합니다.
- **파일 저장 명명 규칙:** 직관적인 구분을 위해 다운로드 파일명은 원본 문자열이 아닌 **"리포트 제목 (Report Title)"** 을 기반으로 강제 포맷팅됩니다 (v1.2 업데이트). 제목 문자열 중 파일 경로로 쓸 수 없는 특수문자(`\`, `/`, `:`, `*` 등)만 치환하며 띄어쓰기는 그대로 유지합니다. 리스트에서 제목 추출이 실패할 경우 상세 페이지 `<title>`에서 참조합니다.
- **다운로드 파일 헤더 검증 (v1.4 신규):** 다운로드 시 웹 서버의 HTTP Response Header(`Content-Disposition`)를 파싱하여 원래 서버가 의도한 파일 확장자(예: `.hwp`, `.zip`)를 보존하여 다운로드합니다.
- **커스텀/페이징 순회 로직 (v1.3 신규):** KDI 및 산업연구원(KIET)과 같이 특정 조건(`pg=` 파라미터, 아코디언 메뉴 대체 `year/month` 변환 등)에 따라 이전 내역을 동적으로 불러와야 하는 사이트들을 위해 자동 페이지 순회 및 한글 복합 날짜 포맷(`YYYY. MM월` 등) 매칭 보정이 적용되었습니다. 
- **비상 텍스트 스크래핑:** 만약 PDF 등을 찾을 수 없다면, 해당 페이지 내 메뉴 바와 스크립트 등을 제거한 순수 `body` 텍스트를 정제하여 `.txt`로 저장합니다. 렌더링 최적화를 위해 CSS와 이미지 로드는 사전에 차단됩니다.
